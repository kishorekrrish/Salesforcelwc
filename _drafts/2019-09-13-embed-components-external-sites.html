Embed Lightning Web Components in External Websites using Lightning Out


To create campaigns and launch surveys or get leads from your community sites, external apps or websites, salesforce has provided a feature called Lightning Out which enables our highly powered lightning components to work seamlessly on external servers/websites.


We are going to use community and guest user


LIVE components on your website in less than 15 minutes.

You can experience Lightning Web Component on this website. Please click on the round blue add button to the bottom right corner, which is nothing but Lightning Web Component. 






Essential requirements to use Lightning Out

  1. We need to have a Lightning component (LWC/AURA).
  2. Lightning App
  3. External Server or Website which can modify both HTML and Javascript.
  4. Communities enabled. (There are different methods, in our method we use communities)
  5. Your website must be whitelisted in your Salesforce org CORS.

Lightning App considerations:

  1. The Lightning App used as Lightning Out container is called Lightning Dependency App. This App doesn't work as a regular App, don't try to preview you will surely get an error.
  2. This App can only be used to specify dependencies for Lightning Out. The access level to be set for the App is GLOBAL.
  3. Most importantly it has to extend "ltng:outApp" and implement "ltng:allowGuestAccess".
  4. Note: Based on the requirement one can extend either "ltng:outApp" or "ltng:outAppUnstyled", as the name suggests "ltng:outAppUnstyled" is used when you want to use your own CSS and not SLDS.
  5. If you thinking of using more Lightning components on the page make sure that, you add that component as a dependency in the Lightning App, Limit is unknown. Child components will be automatically retrieved, No need for explicitly specifying their dependency.

Add Lightning Out JS library to the external website.

This step is important because the external website to run your code it needs to download the framework and resources, which is fulfilled by specifying the JS library.

<script
src="https://myDomain.my.salesforce.com/{community-page-name}/lightning/lightning.out.js">
</script>

don't forget to add your community page domain name and page name in the above URL.







$Lightning.Use()

This function takes four arguments (appName, callback function, lightningEndPointUri, authToken/session-id)

This function loads and initializes the lightning component framework and Lightning components.


$Lightning.createComponent()

As the name suggests, this function allows you to add or create components for which dependency is specified in the App.

arguments (componentName, Attributes, domLocator, authToken/session-id)


Aura Component:

Test.cmp

<aura:component access="global">

 <h1>This is Lightning Component</h1>

</aura:component>

Lightning Web Component:

Test.cmp


<template>

 <h1>This is Lightning Component</h1>

</template>


Lightning Out App:

myFirstApp.app

<aura:application access="GLOBAL" extends="ltng:outApp" implements="ltng:allowGuestAccess"> 
 
   <aura:dependency resource="c:Test"/>

</aura:application>

Embed this script on your external website or server:

<!-- LIVE APP-->
<div id="componentHolder">
</div>
<script src="https://myDomain.my.salesforce.com/{community-page-name}/lightning/lightning.out.js"></script>

<script>
    $Lightning.use("c:myFirstApp", function() {
    $Lightning.createComponent("c:Test",
    { },
   "componentHolder",
    function(component) {
      console.log("LWC component was created");
    });
 },
   'https://myDomain.my.salesforce.com/{community-page-name}',
   'session-id');
 </script>
<!-- LIVE APP-->
